# GitHub 开源发布指南

## 步骤 1：初始化 Git 仓库

在项目根目录执行以下命令：

```bash
cd /Users/chris/new-work/SickYeah-backend

# 初始化 Git 仓库
git init

# 添加所有文件到暂存区
git add .

# 查看将要提交的文件（确认没有敏感信息）
git status

# 创建初始提交
git commit -m "Initial commit: SickYeah Backend API

- Express + TypeScript + Prisma + SQLite
- 用户认证系统 (JWT)
- 餐厅管理 API
- 评价系统
- 文件上传功能
- 完整的 API 文档"
```

## 步骤 2：在 GitHub 上创建仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `SickYeah-backend`（或其他名称）
   - **Description**: `餐厅打卡应用后端 API - Express + TypeScript + Prisma`
   - **Visibility**: 
     - ✅ Public（公开）- 开源项目
     - ⬜ Private（私有）- 私有项目
   - **不要勾选**：
     - ⬜ Add a README file（我们已有）
     - ⬜ Add .gitignore（我们已有）
     - ⬜ Choose a license（我们已有 MIT License）
3. 点击 **Create repository**

## 步骤 3：关联远程仓库并推送

GitHub 会显示类似如下的命令，执行它们：

```bash
# 添加远程仓库（替换为你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/SickYeah-backend.git

# 或使用 SSH（如果已配置 SSH 密钥）
git remote add origin git@github.com:YOUR_USERNAME/SickYeah-backend.git

# 设置默认分支为 main
git branch -M main

# 推送到 GitHub
git push -u origin main
```

1. 点击右上角的 ⚙️ 设置图标

推送成功后：

1. 访问你的 GitHub 仓库页面

## 常见问题

### Q1: 如果不小心提交了敏感文件怎么办？

```bash
# 从 Git 历史中移除文件
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env" \
  --prune-empty --tag-name-filter cat -- --all

# 强制推送（⚠️ 谨慎使用）
git push origin --force --all
```

更好的方式是使用 `git-filter-repo` 或 BFG Repo-Cleaner。

### Q2: 如何更新 .gitignore 后清理已追踪的文件？

```bash
# 清除 Git 缓存
git rm -r --cached .

# 重新添加所有文件
git add .

# 提交更改
git commit -m "Update .gitignore and remove tracked files"
```

### Q3: 如何回滚到之前的提交？

```bash
# 查看提交历史
git log --oneline

# 回滚到指定提交（保留文件更改）
git reset --soft <commit-hash>

# 回滚到指定提交（丢弃文件更改）
git reset --hard <commit-hash>
```

## 后续维护

### 创建开发分支

```bash
# 创建并切换到开发分支
git checkout -b develop

# 推送开发分支到远程
git push -u origin develop
```

### 标记版本

```bash
# 创建版本标签
git tag -a v1.0.0 -m "Release version 1.0.0"

# 推送标签到远程
git push origin v1.0.0

# 推送所有标签
git push origin --tags
```

### 保护主分支

在 GitHub **Settings** → **Branches** 中：

1. 点击 **Add rule**
2. Branch name pattern: `main`
3. 勾选：
   - ✅ Require pull request reviews before merging
   - ✅ Require status checks to pass before merging
